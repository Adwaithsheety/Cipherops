# Screenshots

```markdown
Here's the list of website screenshot tools:

1. [ScreenshotsEyeWitness](https://github.com/FortyNorthSecurity/EyeWitness) - EyeWitness is designed to take screenshots of websites, provide server header info, and identify default credentials if possible.
2. [Aquatone](https://github.com/michenriksen/aquatone) - Aquatone is a tool for visual inspection of websites across a large number of hosts, allowing for a quick overview of the HTTP-based attack surface.
3. [Screenshoteer](https://github.com/vladocar/screenshoteer) - Screenshoteer enables making website screenshots and mobile emulations from the command line.
4. [Gowitness](https://github.com/sensepost/gowitness) - Gowitness is a web screenshot utility written in Go, utilizing Chrome Headless.
5. [WitnessMe](https://github.com/byt3bl33d3r/WitnessMe) - WitnessMe is a web inventory tool that takes screenshots of webpages using Pyppeteer (headless Chrome/Chromium) and offers additional features to simplify the process.
6. [Eyeballer](https://github.com/BishopFox/eyeballer) - Eyeballer is a convolutional neural network for analyzing pentest screenshots.
7. [Scrying](https://github.com/nccgroup/scrying) - Scrying is a tool for collecting RDP, web, and VNC screenshots, consolidating them in one place.
8. [Depix](https://github.com/beurtschipper/Depix) - Depix is a tool for recovering passwords from pixelized screenshots.
9. [HTTPScreenshot](https://github.com/breenmachine/httpscreenshot/) - HTTPScreenshot is a tool for capturing screenshots and HTML from a large number of websites.

```
